二預算架構全貌（最重要考點）▼

Ch7 最核心的概念：預算由下到上一層一層堆疊，每一層的內容和用途都不同。

💰 成本預算 Cost Budget = 成本基準 + 管理儲備（整個專案批准的資金上限）
📊 成本基準 Cost Baseline = 控制帳戶成本 + 應變儲備（PM 的績效監控基準）
🗂️ 控制帳戶 Control Account = 工作包成本 + 規劃包（PM 的主要管控點）
📦 工作包 Work Package = 各活動成本加總（最詳細估算的基本單位）
層級誰用用途
成本預算
Cost Budget		Sponsor / 高階管理層整個專案批准的資金上限，包含管理儲備
成本基準
Cost Baseline		PM績效監控基準，由制定預算流程產出，包含應變儲備但不含管理儲備
控制帳戶
Control Account		PM / 客戶 / SponsorPM 一般狀況估算工期成本的管控點；客戶和 Sponsor 驗收績效基準的主要管控點
工作包
Work Package		PM / 團隊成本估算的基本單位；利害關係人若無法信任工作包估算，可再細分到「活動」層級
⚠️ 最常考：成本基準 vs 成本預算的差異
差異就是「管理儲備」：成本基準 + 管理儲備 = 成本預算

成本基準在 EVM 圖上是下方那條水平線，成本預算是上方那條。
EAC 預測線超過成本基準但低於成本預算 → 還在可接受範圍
EAC 預測線超過成本預算 → 需向 Sponsor 申請追加預算

預算包含「範疇及時程」嗎？→ 是的，預算是根據範疇和時程制定的

「主管要求詳細估算，最準確應估算到哪層？」 → 工作包（Work Package）
「利害關係人無法信任工作包估算，應再估算到哪層？」 → 活動（Activity）（更細一層）
三儲備金：應變儲備 vs 管理儲備（超高頻考點）▼
類型應對什麼風險誰負責如何使用在哪層
應變儲備
Contingency Reserve		已知風險（Known Unknowns）
已識別、已評估的風險		PMPM 可直接使用，不需提出變更申請成本基準內
管理儲備
Management Reserve		未知風險（Unknown Unknowns）
無法預期的突發事件		Sponsor / 高階管理層（擁有者）PM 提出變更請求送 CCB 核准後才能使用，並移入成本基準成本基準外（預算內）
💡 記憶方法
應變儲備：「我知道可能會出這個問題，先準備一筆錢」→ PM 自己管，直接用
管理儲備：「我不知道會出什麼問題，先留一筆備用金」→ Sponsor 擁有，動用要走 CCB

為什麼管理儲備要送 CCB？因為動用管理儲備會將那筆錢移入成本基準，改變了基準，所以必須走正式變更流程。
⚠️ 高頻考題情境
「颱風警報發布，PM 需要採購砂包加固，如何處理？」
颱風 = 可預期的已知風險 → 應變儲備，PM 直接使用

「專案執行中成員辨識出未曾預見的重大事件，PM 應如何？」
未曾預見 = 未知風險 → 管理儲備，提出變更請求送 CCB 核准

「關於成本基準，何者不正確？」
✅ 正確說法（不是答案）：「未知風險用管理儲備，發生時提變更計畫，核准後移入成本基準」
❌ 不正確說法（這才是答案）：「應變儲備 PM 需要提出變更申請才能使用」→ 錯！應變儲備 PM 可直接用

「應變儲備是已知高風險還是已知低風險？」 → 考題選已知高風險
四EVM 實獲值管理（計算題）▼

三個基本數值

縮寫全名意思
PVPlanned Value 計畫值到目前為止，計畫應該完成多少工作（值多少錢）
EVEarned Value 實獲值到目前為止，實際完成了多少工作（值多少錢）
ACActual Cost 實際成本到目前為止，實際花了多少錢
BACBudget at Completion 完工預算整個專案原始核定的總預算

六個核心公式（全部要背）

CV（成本差異 Cost Variance）
CV = EV − AC
正數 = 成本節省（好）｜負數 = 成本超支（不好）
SV（時程差異 Schedule Variance）
SV = EV − PV
正數 = 進度超前（好）｜負數 = 進度落後（不好）
CPI（成本績效指標 Cost Performance Index）
CPI = EV ÷ AC
>1 = 成本效率好｜<1 = 成本超支
SPI（時程績效指標 Schedule Performance Index）
SPI = EV ÷ PV
>1 = 進度超前｜<1 = 進度落後
EAC（完工估算 Estimate at Completion）— 最常考
EAC = BAC ÷ CPI（假設未來績效和目前一樣）
EAC = AC + (BAC − EV)（假設未來績效回到計畫水準）
EAC = AC + [(BAC − EV) ÷ CPI]（假設未來用特定效率完成）
預測最終總花費｜除非題目特別說「未來績效回正常」，否則用第一個公式
ETC（剩餘工作估算）/ VAC（完工差異）
ETC = EAC − AC（還需要花多少錢）
VAC = BAC − EAC（正數=最終節省，負數=最終超支）
💡 EV 是主角——看圖判斷口訣
EV vs PV → 看進度（時程）
EV 在 PV 上方 = 進度超前（SPI > 1）；EV 在 PV 下方 = 進度落後（SPI < 1）

EV vs AC → 看成本
EV 在 AC 上方 = 成本節省（CPI > 1）；EV 在 AC 下方 = 成本超支（CPI < 1）

CPI > 1 → EAC < BAC（最終花費比預算少）
CPI < 1 → EAC > BAC（最終花費比預算多）

看圖判斷的四種情境

圖上看到狀況PM 應如何
EV < PV，AC > EV進度落後 + 成本超支（最糟）SPI<1, CPI<1分析原因，向管理層報告，評估應對方案
EV < PV，AC < EV進度落後 + 成本節省 SPI<1, CPI>1用省下的錢趕工（Crashing）加速進度
EV > PV，AC > EV進度超前 + 成本超支 SPI>1, CPI<1注意成本控制，分析超支原因
EV > PV，AC < EV進度超前 + 成本節省（最好）SPI>1, CPI>1維持現狀，向管理層報告好消息
⚠️ CPI 和 SPI 分母不要搞混
CPI = EV ÷ AC（和錢比，看成本）
SPI = EV ÷ PV（和計畫比，看進度）
分子都是 EV，分母不同。

六題目出題模式與陷阱彙整▼
📝 題型1：「目前在哪個流程？」
  • 「說明計量單位、控制臨界點、績效測量規則」→ 規劃成本管理
  • 「評估自行開發與委外的成本」→ 估算成本
  • 「將成本分派至每項交付成果，建立成本基準」→ 制定預算
  • 「監控成本，用 CPI、EAC 評估」→ 控制成本
📝 題型2：「用哪種儲備金？由誰核准？」
  • 已知/可預期風險發生 → 應變儲備，PM 直接使用
  • 未知/未曾預見風險發生 → 管理儲備，提變更請求送 CCB 核准
  • 颱風（可預期）→ 已知風險 → 應變儲備，PM 直接使用
📝 題型3：EVM 指標判讀
  • CPI < 1 → 成本超支，EAC > BAC
  • CPI > 1 → 成本節省，EAC < BAC
  • SPI > 1 → 進度超前；SPI < 1 → 進度落後
  • 進度落後 + 成本有結餘（CPI>1）→ 用省下的錢 趕工（Crashing）
  • CPI=0.7, SPI=0.8 → 雙重問題 → 向 Sponsor 報告，討論應對策略
📝 題型4：「估算到哪個層級？」
  • 主管要詳細估算 → 工作包（Work Package）
  • 利害關係人無法信任工作包估算 → 活動（Activity）
  • 客戶驗收績效基準的主要管控點 → 控制帳戶（Control Account）
  • 預算編列的基礎 → WBS
💡 Ch7 核心思維
  • 成本超支不是 PM 自己扛，透過 EVM 提早發現，即時向管理層提出 EAC 預測和應對方案
  • 看到「成本超支題目」→ 答案通常是「評估影響、提出方案、向上報告」，不是默默省錢
  • 砍預算 = 縮範疇，PM 要讓 Sponsor 清楚代價
★A卷計算題解析▼
📝 Q138：BAC=$200,000，AC=$160,000，已完成$128,000工作，CPI=？

EV = $128,000（已完成工作的計畫值）、AC = $160,000

CPI = EV ÷ AC = 128,000 ÷ 160,000 = 0.8

→ 成本超支（每花1元只做了0.8元的事）。陷阱：0.85是錯的，0.8才對。

📝 Q140：CPI=2，BAC=$150,000，EAC=？

EAC = BAC ÷ CPI = 150,000 ÷ 2 = $75,000

CPI=2 代表效率超高，最終只需花一半的預算。→ 答案 $75,000

📝 Q152：EV=300，AC=400，CV=？

CV = EV − AC = 300 − 400 = −100

負數代表成本超支 $100。

📝 Q144：SPI=0.75，CPI=0.8，PM 應如何？

SPI=0.75 → 進度落後；CPI=0.8 → 成本超支。兩個都小於 1，雙重問題。

→ 向 Sponsor 報告，一起評估可行的應對方案（不是自己默默趕工，不是縮減範疇）

📝 EAC 情境題：假設未來績效回到正常

當題目說「應對方案核准後，假設未來績效回到計畫水準」：

EAC = AC + (BAC − EV)（不用 BAC÷CPI）
