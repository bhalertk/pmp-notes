一成本管理是什麼？ ▼

成本管理（Cost Management）就是回答：「這個專案要花多少錢？怎麼控制不超支？」

💡 對應您的銀行 AP 專案
銀行系統開發都有預算限制，您應該熟悉：預算是怎麼估出來的？開發途中遇到突發狀況要追加費用怎麼辦？最後結案時實際花了多少？

PMP 把這些問題系統化，特別是用 EVM（實獲值管理） 這套工具，可以隨時知道「目前進度好不好、成本超了沒」。
二成本管理的時間軸：四個流程 ▼
①
規劃成本管理 Plan Cost Management 規劃

制定「如何估算、編列預算、管理、監控成本」的計畫

📄 成本管理計畫（包含：測量單位、控制臨界點、測量績效規則等）
⚠️ 注意：以 WBS 作為基礎進行成本估算、制定及控制預算，並說明預算的計量單位、控制的臨界點與測量績效規則，這些都是「規劃成本管理」的工作
②
估算成本 Estimate Cost 規劃

估算完成每個活動、工作包所需的費用

📄 成本估算（Activity Cost Estimates）
→ 評估手機介面設計的自行開發與委外設計的成本，考量所有資源的相關成本 → 這是「估算成本」
③
制定預算 Determine Budget 規劃

將各活動成本加總，建立成本基準（Cost Baseline）

📄 成本基準 Cost Baseline（用於監控專案績效的基準）
→ 將資源及活動的成本加總至 WBS 工作包，再向上加總到控制帳戶 → 這叫「成本匯總（Cost Aggregation）」，屬於制定預算的技術
→ 取得成本基準必須進行「制定預算」流程
④
控制成本 Control Cost 監控

監控成本執行狀況，管理成本基準的變更，使用 EVM 評估績效

使用實獲值管理（EVM）計算 CPI、SPI、EAC 等指標
💡 詳細時間及成本估算在哪個流程群組進行？
答案：專案規劃階段。起始階段只做高層次估算，詳細估算要等到規劃階段才能進行。
三預算架構全貌（最重要考點） ▼

Ch7 最重要的概念就是這個預算架構，由下到上一層一層堆疊：

💰 成本預算 Cost Budget（最大範圍）= 成本基準 + 管理儲備
📊 成本基準 Cost Baseline= 工作包成本 + 應變儲備（已批准的預算基準，用於績效監控）
🗂️ 控制帳戶成本 Control Account= 工作包成本 + 規劃包 + 應變儲備
📦 工作包成本 Work Package Cost= 各活動成本加總
💡 一句話記住差異
成本基準：PM 負責管理的預算範圍，包含「已知風險的應變儲備」
成本預算：比成本基準再加一層「管理儲備」（未知風險準備金），由 Sponsor 負責

所以：預算 = 成本基準 + 管理儲備
⚠️ 「成本基準與成本預算的差異是什麼？」
答案：差異就是「管理儲備（Management Reserve）」
成本基準 + 管理儲備 = 成本預算

預算包含「範疇及時程」嗎？→ 是的！預算包含範疇、時程及成本（因為預算是根據範疇和時程制定的）

各層級的管理點

層級誰用來做什麼
控制帳戶
Control Account		PM 一般狀況估算工期、成本及資源的管理點；客戶驗收績效基準的主要管控點；Sponsor 驗收管理的點
工作包
Work Package		利害關係人無法信任工作包的估算時，PM 會估算到更細的「活動（Activity）」層級
活動
Activity		主管希望 PM 詳細估算時，最準確的估算應做到「工作包（Work Package）」層級
（注意：工作包比活動更高一層，但最詳細的估算是到活動層）
⚠️ 最容易混淆的考點
「主管希望 PM 詳細估算，應估算到那一層最準確？」
→ 答案：工作包（Work Package），不是活動。Work Package 是 WBS 最底層，是成本估算的基本單位。

「利害關係人無法信任工作包的工期、成本及資源的估算，PM 應估算至哪層？」
→ 答案：活動（Activity），比工作包更細一層，給利害關係人更詳細的數據。
四儲備金：應變儲備 vs 管理儲備（超高頻考點） ▼
類型應對什麼風險誰負責如何使用在哪層
應變儲備
Contingency Reserve		已知風險（Known Unknowns）
已識別、已評估的風險		PMPM 可以直接使用，不需要提出變更申請成本基準內
管理儲備
Management Reserve		未知風險（Unknown Unknowns）
無法預期的突發事件		Sponsor / 高階管理層發生時由 PM 提出申請，獲得核准後才能使用，並移入成本基準成本基準外（預算內）
💡 記憶方法：已知 vs 未知
應變儲備 = 「我知道可能會出這個問題，先準備一筆錢」→ PM 自己管
管理儲備 = 「我不知道會出什麼問題，先留一筆備用金」→ Sponsor 管，用的時候要申請

考題情境：「專案受到颱風影響，通常會用砂包來加固，當颱風警報發布，PM 如何？」
颱風是「已知風險」（事先能預期）→ 用應變儲備，PM 可直接使用，不需送 CCB。
⚠️ 最常答錯的儲備金題目
「專案執行中成員辨識出未曾預見的重大事件，PM 應如何？」
未曾預見 = 未知風險 → 用管理儲備，送給 Sponsor（贊助人）核准，不是送 CCB！

「應變儲備是哪種風險？」→ 已知高風險 or 已知低風險？
應變儲備應對的是「已知的」風險，至於高低都有可能。考題選「已知高風險」。

管理儲備 = 未知成本，先編列預算「發生時才提計畫，計畫核准後才執行」（C選項）
五成本基準 vs 成本預算 ▼
概念組成用途
成本基準
Cost Baseline		工作包成本 + 應變儲備作為監控專案成本績效的基準，由「制定預算」流程產出
成本預算
Cost Budget		成本基準 + 管理儲備整個專案批准的資金上限
⚠️ 關於成本基準的錯誤說法
考題（Q22）：「關於成本基準的敘述，何者不正確？」

❌ 錯誤：「未知風險，由 Sponsor 負責，編列管理儲備；發生時由 PM 提出風險回應計畫，獲得變更核准後，將管理儲備移入成本基準」
→ 這其實是正確的描述！所以這個選項不是「不正確」的那個。

✅ 真正不正確的說法：「已知風險，由 PM 負責，並編列活動或工作包的應變儲備，發生時 PM 需要提出變更申請獲得核准才能使用」
→ 錯誤在於：應變儲備是 PM 可以直接使用的，不需要提出變更申請！
六EVM 實獲值管理（計算題） ▼

EVM（Earned Value Management）是用數字來評估專案「進度好不好、成本超了沒」的工具。Ch7 一定有計算題。

三個基本數值

縮寫全名意思
PVPlanned Value
計畫值		到目前為止，計畫應該完成多少工作（值多少錢）
EVEarned Value
實獲值		到目前為止，實際完成了多少工作（值多少錢）
ACActual Cost
實際成本		到目前為止，實際花了多少錢
BACBudget at Completion
完工預算		整個專案原始核定的總預算

四個核心指標與公式

CV（成本差異 Cost Variance）
CV = EV − AC
正數 = 成本節省（好）｜負數 = 成本超支（不好）
SV（時程差異 Schedule Variance）
SV = EV − PV
正數 = 進度超前（好）｜負數 = 進度落後（不好）
CPI（成本績效指標 Cost Performance Index）
CPI = EV ÷ AC
>1 = 成本效率好（每花1元做了超過1元的事）｜<1 = 成本超支
SPI（時程績效指標 Schedule Performance Index）
SPI = EV ÷ PV
>1 = 進度超前｜<1 = 進度落後｜= 1 = 準時

完工估算 EAC

EAC（完工估算 Estimate at Completion）
EAC = BAC ÷ CPI
預測整個專案最終總花費（假設未來績效和目前相同）

實際計算（Q13）

💡 解題步驟
題目：BAC = 10萬元，CPI = 0.9，請問 EAC 如何？

EAC = BAC ÷ CPI = 10萬 ÷ 0.9 = 11.1萬

CPI = 0.9 代表「每花1元只做了0.9元的事」，成本效率不好，所以最終預計會超支，EAC 大於 BAC（10萬）。
→ 答案選「大於10萬」。
💡 快速判斷：不用計算的 EAC 方向判斷
CPI > 1 → 成本效率好 → EAC < BAC（最終花費比預算少）
CPI < 1 → 成本超支 → EAC > BAC（最終花費比預算多）
CPI = 1 → 剛好 → EAC = BAC

專案進度落後、成本有結餘的情況

💡 「一個專案進度落後，成本有結餘，PM 該如何？」
進度落後（SPI < 1）但成本有結餘（CPI > 1）→ 代表做得比較慢但比較省錢。
PM 的正確做法：趕工（Crashing），用省下的錢來加速進度，在不增加總預算的情況下提前完成。
七估算方法與管理層級 ▼

成本估算方法（與時程估算相同）

方法特點考題關鍵字
類比估算
Analogous		參考過去類似專案，快速但準確度低「以前多個案子的資料，快速估算整體工期」
參數估算
Parametric		單位 × 數量，有公式計算，較準確「統計分析取得製程及材料單價」、「每坪×坪數」、「每模組美元數」
三點估算
PERT		(O+4M+P)÷6，資料分散時用「資料非常分散」、「不確定性高」
由下而上估算
Bottom-up		最詳細準確，從最小單位向上加總「最詳細、最準確」
⚠️ Sponsor 不滿意估算，要求砍10%，PM 應如何？
考題（Q7）：「Sponsor 對提供的成本估算不滿意，要求砍掉10%成本估算」

❌ 錯誤做法：要求團隊直接將估計值降低10%、開始尋求節省費用的方法
✅ 正確做法：與 Sponsor 討論，說明可能有部分工作範圍無法納入

成本估算是根據範疇算出來的，直接砍預算等於縮減範疇，要讓 Sponsor 清楚這個代價。

供應商提出成本增加，PM 應如何？

💡 正確做法
考題（Q12）：「供應商告訴你，你的專案活動增加了他們的成本」
→ 正確做法：向該供應商詢問更進一步的細部資訊（先了解原因）
→ 不是直接否認、不是立刻召開變更管理會議、不是直接提出變更成本的建議
八題目出題模式與陷阱彙整 ▼
📝 題型1：「目前在哪個流程？」
  • 「評估自行開發與委外的成本，考量所有資源相關成本」→ 估算成本
  • 「將成本分派至每項交付成果，建立成本基準」→ 制定預算
  • 「以 WBS 作為基礎進行成本估算、制定及控制預算，說明計量單位、控制臨界點」→ 規劃成本管理
  • 「監控成本績效，用 CPI、EAC 評估」→ 控制成本
📝 題型2：「用哪種儲備金？由誰核准？」
  • 已知風險發生 → 應變儲備，PM 直接用，不需申請
  • 未知/未曾預見的風險發生 → 管理儲備，送 Sponsor 核准
  • 颱風（事先可預期）→ 已知風險 → 應變儲備，PM 直接使用
📝 題型3：EVM 指標判讀
  • CPI = 0.9 → 成本超支，EAC 大於 BAC
  • CPI = 1.1 → 成本節省，EAC 小於 BAC
  • SPI > 1 → 進度超前；SPI < 1 → 進度落後
  • 進度落後 + 成本有結餘 → 用省下的錢 趕工（Crashing）
📝 題型4：「估算到哪個層級最準確？」
  • 主管要詳細估算 → 工作包（Work Package）
  • 利害關係人無法信任工作包估算 → 活動（Activity）（更細一層）
  • 客戶驗收績效基準的主要管控點 → 控制帳戶（Control Account）
  • 預算是以什麼為基礎編列？→ WBS（工作分解結構）
⚠️ 最常答錯的考點
「應變儲備是已知還是未知風險？」
→ 應變儲備 = 已知風險（Known Unknowns）
→ 管理儲備 = 未知風險（Unknown Unknowns）

「應變儲備發生時，PM 需要提出變更申請才能使用」→ 錯！
應變儲備 PM 可直接使用；要提申請才能用的是管理儲備。

「成本基準和成本預算的差異是什麼？」→ 管理儲備

歷屆題型有多題是「給你一張 EVM 圖，問你現在專案狀況如何、PM 應如何？」，不只考公式，要能看圖說故事。

標準 EVM 圖的組成

線條代表特徵

AC 線（實際成本）實際花了多少錢和 EV 比較看成本效率

圖上看到代表CPI/SPIPM 應如何
EV 低於 PV
AC 高於 EV		進度落後 + 成本超支（最糟）SPI < 1
CPI < 1		立即分析原因，向管理層報告，評估趕工或縮減範疇
EV 低於 PV
AC 低於 EV		進度落後 + 成本節省SPI < 1

CPI > 1		維持現狀，向管理層報告好消息
💡 看圖的口訣
EV vs PV → 看進度（時程）
EV 在 PV 上方 = 進度超前（SPI > 1）
EV 在 PV 下方 = 進度落後（SPI < 1）

EV vs AC → 看成本
EV 在 AC 上方 = 成本節省（CPI > 1，做了比花的多）
EV 在 AC 下方 = 成本超支（CPI < 1，做的沒有花的多）

記憶方法：EV 是主角，EV 和誰比就看哪個指標。

Management Reserve（管理儲備）在圖上的位置

💡 Cost Budget vs Cost Baseline 在圖上的差異
圖上通常會看到兩條水平線：
上方那條 = Cost Budget（成本預算）= 包含管理儲備
下方那條 = Cost Baseline（成本基準）= 不含管理儲備

EAC 預測線若超過 Cost Baseline 但低於 Cost Budget → 還在可接受範圍內
EAC 預測線若超過 Cost Budget → 需要向 Sponsor 申請追加預算
⚠️ 歷屆考題情境
「圖上顯示 AC 一直高於 EV，EV 也低於 PV，PM 應如何？」
→ 進度落後且成本超支 → 立即分析原因，評估影響，向管理層提出應對方案
→ 不是默默繼續工作，不是直接縮減範疇，要先分析原因再提方案

「CPI = 0.7，SPI = 0.8，PM 最好應做什麼？」
→ 兩個都小於 1，情況嚴重 → 向 Sponsor 報告，討論可行的應對策略
★A卷補強：計算題全解析▼

EVM 公式快速複習

CV（成本差異）
CV = EV − AC
正數=節省 負數=超支
SV（時程差異）
SV = EV − PV
正數=超前 負數=落後
CPI（成本績效）
CPI = EV ÷ AC
>1好 <1超支
SPI（時程績效）
SPI = EV ÷ PV
>1超前 <1落後
EAC（完工估算）
EAC = BAC ÷ CPI
預測最終總花費
ETC（剩餘工作估算）
ETC = EAC − AC
還需要花多少錢
VAC（完工差異）
VAC = BAC − EAC
正數=最終會節省 負數=最終會超支

A卷實際計算題解析

📝 Q138：BAC=$200,000，AC=$160,000，已完成$128,000工作，CPI=？

已知：EV = $128,000（已完成的工作值多少）、AC = $160,000（實際花了多少）

CPI = EV ÷ AC = 128,000 ÷ 160,000 = 0.8

→ 答案 B（0.85 是陷阱，0.8 才對）

📝 Q139：BAC=$200,000，EV=$128,000，AC=$160,000，SPI=？

SPI = EV ÷ PV，但題目沒給 PV？→ 要看題目敘述，PV 通常由「計畫完成的工作量」算出

如果 SPI=0.85，代表：EV = PV × 0.85，進度落後 15%

⚠️ 您答錯的關鍵
CPI 和 SPI 的分子都是 EV，但分母不同：
CPI = EV ÷ AC（和錢比）
SPI = EV ÷ PV（和計畫比）
看題目問的是成本還是時程，再對應分母。
📝 Q140：CPI=2，BAC=$150,000，SPI=1，EAC=？

EAC = BAC ÷ CPI = 150,000 ÷ 2 = $75,000

CPI=2 代表每花 1 元做了 2 元的事，效率超高，最終只需花一半的預算。

→ 答案 B（$75,000）

📝 Q152：EV=300，AC=400，BAC=$325，CV=？

CV = EV − AC = 300 − 400 = −100

負數代表成本超支 $100。

→ 答案 D（-100）

📝 Q144：SPI=0.75，CPI=0.8，專案目前狀況和 PM 應如何？

SPI=0.75 → 進度落後（完成了計畫的 75%）
CPI=0.8 → 成本超支（每花 1 元只做了 0.8 元的事）

兩個都小於 1，是雙重問題。PM 應該：向 Sponsor 報告，一起評估可行的應對方案（不是自己默默趕工，不是縮減範疇）

📝 Q158：EAC預測線到 $1,000來算，SPI=1，Sponsor 批准了 PM 建議的應對方案，CPI 會是多少？

這類題型的邏輯：SPI=1 代表進度剛好，成本如果改善後 CPI 回到 1，則 EAC = BAC。

關鍵：當 SPI=1 且應對後假設未來績效正常，EAC = AC + (BAC - EV)

成本基準層次計算（Q146、Q162、Q163）

層次計算方式
Activity Cost最底層的活動成本
Work Package Cost= 活動成本加總
Control Account= Work Package + Planning Package
Cost Baseline（成本基準）= Control Account + Contingency Reserve（應變儲備）
Cost Budget（成本預算）= Cost Baseline + Management Reserve（管理儲備）
⚠️ 最常混淆：誰在 Baseline 內？誰在外？
應變儲備（Contingency Reserve）在 Cost Baseline 內，PM 可直接動用
管理儲備（Management Reserve）在 Cost Baseline 外，需 Sponsor 核准才能動用

Cost Baseline 是監控績效的基準，包含應變儲備但不含管理儲備。

EAC 的不同計算方式（Q159、Q161）

情境公式
假設未來績效和目前一樣EAC = BAC ÷ CPI（最常考）
假設未來績效會回到計畫水準EAC = AC + (BAC − EV)
假設未來用特定效率完成EAC = AC + [(BAC − EV) ÷ CPI]
💡 考試大多數情況用第一個公式
除非題目特別說「假設未來績效回到正常」，否則預設用 EAC = BAC ÷ CPI。
九核心思維總結 ▼
💡 Ch7 成本管理的核心思維
  • 預算架構有層次：Work Package → Control Account → Cost Baseline（含應變儲備）→ Cost Budget（含管理儲備）
  • 兩種儲備金：已知風險用應變儲備（PM 管）；未知風險用管理儲備（Sponsor 管）
  • EVM 是健康指標：CPI、SPI 大於1是好的；EAC = BAC ÷ CPI
  • 預算不能直接砍：成本估算根據範疇，砍預算等於縮範疇，要讓 Sponsor 清楚代價
  • 詳細估算在規劃階段，起始階段只做粗略估算
💡 對應您的實戰經驗
銀行 IT 專案一定有預算限制，您應該很熟悉「預算超了怎麼辦」的壓力。

PMP 的思維是：超支不是 PM 自己扛，而是透過 EVM 提早發現問題，即時向管理層提出預測（EAC）和應對方案，讓決策者做決定。

考試記得：看到成本超支的題目，答案通常是「評估影響、提出方案、向上報告」，不是自己默默想辦法省錢。
